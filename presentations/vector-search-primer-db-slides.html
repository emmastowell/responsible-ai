Under the hood

How Vector Search finds neighbours.

Databricks Vector Search runs an approximate-nearest-neighbour (ANN) search built on HNSW — Hierarchical Navigable Small World graphs. Instead of comparing your query to every vector, HNSW builds a layered proximity graph: search enters at a sparse top layer and hops greedily to ever-closer neighbours as it descends, reaching the nearest points in roughly logarithmic time at high recall. Standard endpoints hold the graph in memory for millisecond latency; storage-optimized endpoints keep it on disk to scale past a billion vectors. Distance metrics: L2 and cosine.

ApproachHow it worksStrengthWeakness
Brute-force (Flat)Compare the query to every vectorExact, zero tuningO(N) per query; won't scale
HNSW ★ DatabricksLayered proximity graph, greedy descentFast, with high recallMemory-hungry; slower to build & update
IVF / IVF-PQCluster vectors into cells, probe the nearestScales; low memory with PQRecall depends on cells probed; PQ is lossy
DiskANNProximity graph stored on SSDBillions of vectors on modest RAMHigher latency; more complex
LSHHash so similar vectors collideSimple; streaming-friendlyLower recall; needs many hash tables

All approximate methods trade a little recall for a lot of speed — Databricks picks HNSW for the standard sweet spot (high recall at millisecond latency), with a disk-based variant for billion-scale.

One shared, governed index

Set up once, queried by everyone.

  • Shared & read-only — one always-on endpoint for the room. You query; you create and delete nothing.
  • Self-managed vectors — a Delta Sync index over the tutorial's stored embedding column. No re-embedding.
  • Same model always — queries must use the same databricks-bge-large-en, or they land in a different vector space.
  • Governed by Unity Catalog — metadata filters at query time; table permissions inherited.
Why this way

The responsible defaults, baked in.

One index instead of dozens keeps cost down; reusing vectors avoids paying twice; one model keeps results comparable.

Clustering vs retrieval.

KMeans clusteringVector search retrieval
Question"What groups exist across the corpus?""What's most similar to this item?"
OutputOne label per row (k=3)Ranked top-k neighbours, with scores
WhenOnce, offline, on the full setOn demand, per query
